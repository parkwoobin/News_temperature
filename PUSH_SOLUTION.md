# GitHub 푸시 문제 해결 방법

## 문제
모델 파일이 너무 커서 (2GB+) GitHub 푸시 시 타임아웃 발생

## 해결 방법 옵션

### 옵션 1: 모델 파일 제외하고 코드만 푸시 (가장 간단) ⭐

배포 플랫폼에서 빌드할 때 모델 파일을 다운로드하도록 설정

**장점:**
- 빠른 푸시
- GitHub 저장소 크기 절약
- 배포 플랫폼에서 자동 다운로드

**단점:**
- 모델 파일을 별도로 관리해야 함

### 옵션 2: Git LFS 사용 (권장)

대용량 파일을 Git LFS로 관리

**단계:**
1. 기존 커밋에서 모델 파일 제거
2. Git LFS로 다시 추가
3. 푸시

### 옵션 3: 모델 파일을 별도 스토리지에 저장

Google Drive, AWS S3 등에 모델 저장 후 런타임에 다운로드

## 추천: 옵션 1 (간단하고 빠름)

코드만 푸시하고, 배포 플랫폼에서 모델을 다운로드하도록 Dockerfile 수정
